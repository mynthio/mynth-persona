# Contributing to Mynth Persona

Thank you for your interest in contributing to Mynth Persona! We welcome contributions from the community to help improve the project.

## Getting Started

1.  **Fork the repository** and clone it to your local machine.
2.  Follow the setup instructions in the [README.md](README.md) to get your local environment running.
3.  Ensure you have all the required API keys set up in your `.env.local` file.

## Development Workflow

1.  **Create a Branch**: Always create a new branch for your changes.
    ```bash
    git checkout -b feature/your-feature-name
    ```
2.  **Make Changes**: Implement your feature or fix.
3.  **Test**: Verify your changes locally. Ensure the app runs without errors and the new functionality works as expected.
4.  **Lint & Format**: Run linting to ensure code quality.
    ```bash
    pnpm lint
    ```

## Pull Request Process

1.  Push your branch to your fork.
2.  Open a Pull Request (PR) against the `main` branch of the original repository.
3.  Provide a clear description of your changes and the problem they solve.
4.  Wait for review. We use CodeRabbit and manual reviews to ensure code quality.

## Code Style

- We use **TypeScript** and **Next.js**.
- Styling is done with **Tailwind CSS**.
- We use **shadcn/ui** for components.
- Follow the existing code style and conventions.
